# Docs

Kairos is archived. These docs are preserved as reference material for the experiment and the design work it explored.

Read these in this order:

1. [Setup](./setup.md)
   For integrating Kairos into an application, defining workers, registering config, and starting supervision.
2. [Architecture](./architecture.md)
   For understanding the runtime topology, module boundaries, and how the supervision tree is split.
3. [Features](./features/README.md)
   For learning one capability at a time, such as enqueueing, scheduling, retries, cancellation, and stale recovery.
4. [Project Status](./project-status.md)
   For the current scope, stability expectations, and what Kairos is intentionally not trying to do yet.

These pages describe the archived `main` branch behavior and module layout.
